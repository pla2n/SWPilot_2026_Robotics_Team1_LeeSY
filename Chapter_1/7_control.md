# [과제 7] 로봇 자동화 제어 공학: 피드백 및 PID 제어 시스템 명세서

## 1. 제어 시스템의 기초 (Introduction to Control Systems)

### 1.1. 제어 시스템의 정의

제어 시스템(Control System)이란 대상 물리계(Plant)의 상태나 출력 변수가 우리가 원하는 목표 상태(Target State)를 추종하도록 적절한 제어 입력(Control Input)을 인가하고 조절하는 하드웨어 및 소프트웨어의 유기적 결합체를 의미합니다.

### 1.2. 제어의 궁극적 목표

자동화 공정 및 로봇 운반차(AGV/AMR) 제어 시스템이 지향하는 핵심 성능 지표는 다음과 같습니다.

- **안정성 (Stability):** 외란이나 초기 오차가 존재하더라도 시스템이 발산하지 않고 일정한 값으로 수렴하는 능력 (제어 시스템의 최우선 조건).
- **속응성 (Responsiveness):** 목표값이 변경되었을 때 얼마나 빠르게 새로운 목표값에 도달하는가를 나타내는 성능 (과도 상태 시간 단축).
- **정밀성 (Accuracy):** 목표값과 최종 수렴한 실제 출력값 사이의 정상상태 오차(Steady-State Error)를 최소화하는 능력.
- **강인성 (Robustness):** 로봇의 적재 하중 변화, 지면 마찰력 변동, 센서 노이즈 등 시스템 파라미터가 불규칙하게 변하더라도 제어 성능을 일정하게 유지하는 능력.

## 2. 피드백 제어의 기본 개념 (Core Concepts of Feedback Control)

피드백 제어(Feedback Control)는 출력단에서 측정된 실제 값을 입력단으로 다시 보내어(Feedback), 목표값과의 비교를 통해 시스템을 수정 제어하는 폐루프 메커니즘입니다.

- **목표값 (Setpoint, $r(t)$):** 사용자가 로봇에게 지시하는 원하는 상태입니다. (예: 주행 속도 100mm/s, 라인의 정중앙 좌표 0)
- **출력값 (Process Variable, $y(t)$):** 엔코더나 IR 센서 등을 통해 실시간으로 측정된 로봇의 실제 물리량 상태입니다. (예: 현재 속도 92mm/s, 현재 라인 치우침 오차 +15mm)
- **오차 (Error, $e(t)$):** 목표값과 실제 출력값 사이의 대수적 차이입니다. 제어기는 이 오차를 실시간으로 모니터링하며 오차가 0이 되도록 동작합니다.
  $$e(t) = r(t) - y(t)$$

## 3. 제어 시스템의 루프 구조 비교 (Open-loop vs Closed-loop)

### 3.1. 오픈 루프 제어 시스템 (Open-loop Control)

[Image of open loop control system block diagram]

- **개념:** 센서를 통한 출력측 피드백 없이, 오직 입력 신호와 제어기 고유의 모델 수식에만 의존하여 액추에이터를 구동하는 방식입니다.
- **특징:** 구조가 단순하고 비용이 저렴하지만, 외부 외란(하중 변화, 경사로, 미끄러짐)에 대응할 수 없습니다.
- **예시:** 모터에 무조건 PWM 50%만 주면 100mm/s로 달릴 것이라 가정하고 굴리는 주행 제어 (배터리가 방전되거나 짐을 실으면 속도가 급감해도 제어기는 인지하지 못함).

### 3.2. 클로즈드 루프 제어 시스템 (Closed-loop / Feedback Control)

- **개념:** 출력값을 센서로 상시 측정하여 제어기로 피드백하고, 오차($e(t)$)에 기반하여 실시간으로 출력을 보정하는 제어 루프입니다.
- **시스템 구성 요소:**
  1. **제어기 (Controller):** 오차를 입력받아 수학적 알고리즘(PID 등)을 거쳐 제어 명령을 연산합니다. (예: MCU, PLC)
  2. **액추에이터 (Actuator):** 제어 명령을 받아 물리적인 일(에너지 변환)을 수행합니다. (예: 모터 드라이버 및 DC 모터)
  3. **대상 시스템 (Plant/Process):** 제어하고자 하는 물리적 대상 자체입니다. (예: 로봇 운반차 본체의 동역학)
  4. **센서 (Sensor):** 플랜트의 실제 상태를 측정하여 전기 신호로 피드백합니다. (예: 엔코더, IR 센서 어레이)

## 4. PID 제어의 개념 및 수학적 표현 (PID Control Theory)

PID 제어는 오차 신호 $e(t)$를 기반으로 **비례(Proportional), 적분(Integral), 미분(Derivative)** 연산을 수행한 값을 합산하여 최종 제어 출력을 생성하는 가장 널리 쓰이는 선형 제어 알고리즘입니다.

### 4.1. 연속 시간 영역에서의 수학적 수식

$$u(t) = K_p e(t) + K_i \int_{0}^{t} e(\tau) d\tau + K_d \frac{de(t)}{dt}$$

- $u(t)$: 제어 출력 (모터 드라이버로 전달될 최종 PWM 전압 등)
- $K_p, K_i, K_d$: 각각 비례, 적분, 미분 제어 이득(Gain, 가중치 상수)

### 4.2. P, I, D 각 항목의 물리적 의미와 역할

#### A. 비례 제어 (Proportional Control : P)

- **역할:** "현재 발생하는 오차의 크기에 비례하여 힘을 제어"합니다.
- **특성:** 오차가 크면 강한 복원력을 내어 목표값으로 빠르게 접근(속응성 향상)시키지만, 오차가 줄어들수록 제어 입력도 함께 작아지기 때문에, 시스템 마찰력과 평형을 이루는 지점에서 더 이상 목표값에 도달하지 못하는 **정상상태 오차(Steady-State Error, 잔류편차)**를 필연적으로 남깁니다. $K_p$가 과도하게 크면 오버슛(Overshoot)이 발생하고 시스템이 진동(Oscillation)하며 발산합니다.

#### B. 적분 제어 (Integral Control : I)

- **역할:** "과거부터 누적된 오차의 역사를 합산하여 제어"합니다.
- **특성:** 미세한 정상상태 오차가 장시간 지속되면 적분 값이 점점 커져 액추에이터를 강하게 밀어붙입니다. 이를 통해 P 제어가 해결하지 못한 **정상상태 오차를 완전히 제거(정밀성 확보)**합니다. 단, 과거 오차가 누적되어 반영되므로 시스템의 반응이 둔해지고 과도한 오버슛을 유발하여 시스템의 안정성을 저해할 수 있습니다. (※ 실무 구현 시 적분 값이 한계치 이상으로 무한히 커져 제어가 불능 상태가 되는 '적분 포화(Integral Windup)' 방지 로직이 필수적입니다.)

#### C. 미분 제어 (Derivative Control : D)

- **역할:** "미래의 오차 변화율을 예측하여 제동(Damping)을 제어"합니다.
- **특성:** 오차가 급격하게 변할 때(오차가 줄어드는 속도가 너무 빠를 때) 그 변화에 저항하는 반대 방향의 제어 입력을 발생시켜 **오버슛을 억제하고 시스템을 안정화**하는 브레이크 역할을 수행합니다. 과도 상태의 진동을 감쇄시켜 정착 시간(Settling Time)을 줄여줍니다. 단, 고주파 센서 노이즈가 유입되면 미분 값이 극단적으로 튀어 시스템에 심한 진동을 줄 수 있으므로 LPF(Low Pass Filter)와 병행해야 합니다.

### 4.3. 임베디드 소프트웨어 구현을 위한 이산화(Discrete) 수식

마이크로컨트롤러(MCU)는 디지털 장치이므로 연속적인 미적분을 수행할 수 없습니다. 따라서 일정 제어 주기($\Delta t$)마다 샘플링되는 이산 데이터 형태로 변환해야 합니다. 현재 주기를 $k$번째 샘플이라 할 때 수식은 다음과 같습니다.

$$u[k] = K_p e[k] + K_i \sum_{n=0}^{k} \left( e[n] \cdot \Delta t \right) + K_d \frac{e[k] - e[k-1]}{\Delta t}$$

## 5. PID 파라미터 튜닝 가이드 (PID Tuning Methods)

실무 제어 엔지니어링에서 가장 중요한 작업은 플랜트 특성에 맞는 최적의 이득 상수($K_p, K_i, K_d$) 조합을 찾아내는 것입니다.

### 5.1. PID 파라미터 증감에 따른 응답 특성 영향도 요약

| 제어 파라미터  | 상승 시간 (Rise Time) | 오버슛 (Overshoot) | 정착 시간 (Settling Time) | 정상상태 오차 (S-S Error) | 안정성 (Stability) |
| :------------: | :-------------------: | :----------------: | :-----------------------: | :-----------------------: | :----------------: |
| **$K_p$ 증가** |      감소 (빠름)      |        증가        |        미미한 변화        |           감소            |        악화        |
| **$K_i$ 증가** |      감소 (빠름)      |        증가        |           증가            |   **제거 (0으로 수렴)**   |        악화        |
| **$K_d$ 증가** |      미미한 변화      |  **감소 (억제)**   |     **감소 (안정화)**     |        미미한 변화        |   개선 (과도 시)   |

### 5.2. 실무 시행착오법 (Trial & Error Tuning) 가이드 라인

자동화 장비 및 물류 로봇 셋업 시 가장 보편적으로 사용되는 직관적 튜닝 순서입니다.

1. **초기화:** $K_p, K_i, K_d$ 값을 모두 0으로 설정합니다.
2. **비례 제어($K_p$) 튜닝:** 목표 계단 입력(Step Input)을 주고, $K_p$ 값을 서서히 올립니다. 목표값에 빠르게 도달하되, 정상상태 근처에서 완만하게 지속적인 진동(Oscillation)이 시작되기 직전 단계의 약 50%~60% 수준으로 $K_p$를 고정합니다.
3. **미분 제어($K_d$) 튜닝:** P 제어로 인해 발생한 오버슛과 정착 제동 성능을 개선하기 위해 $K_d$ 값을 올립니다. 오버슛이 깔끔하게 감쇄되고 부드러운 감속이 이루어질 때까지 조정합니다.
4. **적분 제어($K_i$) 튜닝:** 정상상태에 도달한 후 미세하게 남아있는 편차(잔류 오차)를 없애기 위해 $K_i$ 값을 아주 미세한 단위부터 서서히 올립니다. 잔류 편차가 완전히 사라지고 목표값에 정밀하게 수렴하는 최적 지점을 찾으면 전체 튜닝을 종료합니다.
