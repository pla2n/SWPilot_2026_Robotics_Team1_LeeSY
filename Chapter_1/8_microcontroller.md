# [과제 8] 로봇 제어 시스템 임베디드 공학: 마이크로컨트롤러 및 통신 명세서

## 1. 마이크로컨트롤러(MCU)의 정의 및 역할

### 1.1. 마이크로컨트롤러(Microcontroller Unit, MCU)의 정의

마이크로컨트롤러는 단일 집적 회로(칩) 내에 프로세서 코어(CPU), 메모리(Flash, RAM), 그리고 프로그램 가능한 입출력 주변장치(타이머, ADC, 통신 모듈 등)를 모두 집약시킨 초소형 컴퓨터입니다. 범용 PC의 CPU와 달리, 특정한 하드웨어 시스템을 제어하는 임베디드 시스템(Embedded System)의 두뇌 역할을 수행하도록 설계되었습니다.

### 1.2. 로봇 제어 시스템에서의 핵심 역할

- **센서 데이터 수집 및 전처리:** IR 센서의 디지털 명암 신호, 엔코더의 고속 위상 펄스, IMU의 가속도/각속도 등 물리적인 센서 신호를 하드웨어 핀으로 입력받아 디지털 연산 데이터로 변환합니다.
- **실시간 제어 알고리즘 연산:** 입력된 데이터를 기반으로 순방향/역방향 기구학 수식을 풀거나 PID 제어 루프를 실시간(Real-time) 주기로 연산하여 오차를 수정합니다.
- **액추에이터 구동 신호 생성:** 연산된 제어 출력값을 모터 드라이버가 이해할 수 있는 PWM(펄스 폭 변조) 신호나 방향 전환 신호로 변환하여 출력단에 인가합니다.

---

## 2. 기본 제어 방식과 하드웨어 인터페이스

### 2.1. 디지털 입출력 핀 (Digital I/O Pins)

- **디지털 입력 (Input):** 외부 스위치나 이진화된 센서의 전압 레벨이 0V(LOW)인지 5V/3.3V(HIGH)인지 판별하여 로직 상태를 읽어옵니다.
- **디지털 출력 (Output):** MCU 내부 상태에 따라 핀의 전압을 물리적인 HIGH 또는 LOW로 출력합니다. 이를 통해 모터 드라이버의 정회전/역회전 H-Bridge 방향 스위치를 제어합니다.

### 2.2. 아날로그 인터페이스: ADC (Analog-to-Digital Converter)

- **개념:** 자연계의 연속적인 아날로그 전압 신호(예: 반사 광량에 따라 부드럽게 변하는 포토트랜지스터의 출력 전압)를 MCU가 처리할 수 있는 이산적인 디지털 값으로 변환하는 장치입니다.
- **성능 지표:** 10비트 해상도의 ADC의 경우, 0V ~ 5V의 전압 범위를 1024 단계(0~1023)로 쪼개어 정밀하게 인식할 수 있게 합니다.

### 2.3. PWM (Pulse Width Modulation, 펄스 폭 변조) 출력

- **개념:** 디지털 핀의 HIGH 상태와 LOW 상태의 시간적 비율(Duty Cycle)을 매우 고속으로 ON/OFF 변조하여, 결과적으로 아날로그 전압의 가변 출력과 동일한 효과를 내는 제어 방식입니다.
- **응용:** 모터 드라이버에 인가되는 PWM의 듀티 사이클을 가변함으로써, DC 모터의 평균 공급 전압을 선형적으로 제어하여 부드러운 가감속 주행을 구현합니다.

## 3. 마이크로컨트롤러의 핵심 직렬 통신 프로토콜 비교

### 3.1. UART (Universal Asynchronous Receiver-Transmitter)

- **하드웨어 구성:** TX(송신), RX(수신), GND (총 3선 필요).
- **통신 방식:** 비동기식(Asynchronous) 통신으로, 공통 클럭(Clock) 선 없이 송수신측이 사전에 동일한 통신 속도(Baud Rate, 예: 115200 bps)를 약속하고 패킷을 교환합니다. 1:1 점대점 통신만 가능하며 전이중(Full-Duplex) 통신을 지원합니다.

### 3.2. I2C (Inter-Integrated Circuit)

- **하드웨어 구성:** SDA(데이터), SCL(클럭), GND (총 2선).
- **통신 방식:** 동기식(Synchronous) 통신으로, 고유의 7비트 슬레이브 주소(Address) 제어 방식을 사용하여 단 2개의 선만으로 하나의 마스터에 최대 127개의 센서/장치를 병렬로 연결할 수 있습니다. 반이중(Half-Duplex) 방식으로 동작합니다.

### 3.3. SPI (Serial Peripheral Interface)

- **하드웨어 구성:** MOSI(마스터 출력), MISO(마스터 입력), SCK(클럭), SS/CS(슬레이브 선택), GND.
- **통신 방식:** 주소 지정 오버헤드가 없으며, 독립된 송수신 선과 SS(Slave Select) 선을 이용하여 고속 전이중(Full-Duplex) 데이터 교환을 수행합니다. 세 프로토콜 중 가장 고속(수 MHz 이상) 전송이 가능합니다.

---

## 4. 실시간 제어(Real-time Control)와 타이머 인터럽트

### 4.1. 타이머와 인터럽트의 필요성

로봇 운반차의 PID 제어 알고리즘이나 엔코더 기반 오도메트리 연산은 루프 문 내의 `delay()` 함수와 같은 소프트웨어적 대기 시간에 의해서 주기가 흔들리면 안 됩니다. 제어 주기가 일정하지 않으면 미분/적분 값에 치명적인 오차가 누적되어 제어계가 발산합니다. 하드웨어 타이머는 CPU 내부 클럭을 독립적으로 카운트하여 메인 루프와 상관없이 정확한 시간 주기마다 인터럽트를 발생시켜 일정한 샘플링 주기를 강제로 사수합니다.

## 5. [평가문항 확인] 문제 이해 및 검증 답변

### Q1. 마이크로컨트롤러는 어떤 장치이며, 일반적인 컴퓨터와 비교했을 때 차이점은 무엇인가?

- **답변:** 마이크로컨트롤러(MCU)는 단일 실리콘 칩 내에 CPU, 하드웨어 메모리(Flash/RAM), 입출력 인터페이스(GPIO, 타이머, ADC 등)를 모두 집적한 원칩 컴퓨터입니다. 범용 컴퓨터(PC)가 고성능 CPU, 대용량 RAM, 독립된 GPU 및 스토리지 등을 분리하여 다중 작업(Multi-tasking)과 고속 연산을 처리하는 것과 달리, MCU는 특정 시스템 제어(Single-tasking)에 최적화되어 있어 저전력, 소형화, 그리고 하드웨어 핀 제어에 특화되어 있다는 차이점이 있습니다.

### Q2. 마이크로컨트롤러가 로봇 제어에 주로 사용되는 이유를 설명하라.

- **답변:** 로봇 제어는 센서 신호 유입과 모터 구동 사이의 지연 시간(Latency)이 극도로 짧아야 하는 **실시간성(Real-time Property)**이 필수적입니다. 일반 PC의 OS(Windows, Linux 등)는 스케줄링 연산 오버헤드로 인해 정밀한 제어 주기를 보장하지 못하지만, MCU는 하드웨어 레벨의 인터럽트와 타이머를 제어할 수 있어 마이크로초 단위의 결정론적(Deterministic) 제어가 가능합니다. 또한 PWM, ADC, UART 등 다양한 하드웨어 인터페이스를 칩 자체에 내장하고 있어 기구 제어에 완벽히 부합합니다.

### Q3. 마이크로컨트롤러의 디지털 핀과 아날로그 핀은 어떤 차이가 있으며, 각각 어떤 용도로 사용될 수 있는가?

- **답변:** \* **디지털 핀:** 0(LOW, 0V)과 1(HIGH, 5V 또는 3.3V)의 불연속적인 이진 신호만 처리합니다. 용도로는 버튼/스위치 상태 입력, LED On/Off 제어, 모터 드라이버의 H-Bridge 정회전/역회전 방향 선택 핀 등이 있습니다.
  - **아날로그 핀:** 시간과 전압에 따라 연속적으로 변화하는 전압 신호를 처리합니다. 내장된 ADC(Analog-to-Digital Converter)를 통해 아날로그 전압을 이산 디지털 수치로 변환하며, 용도로는 적외선(IR) 센서의 반사 광량 아날로그 값 읽기, 온도/습도 센서 측정, 조이스틱 전압 측정 등에 활용됩니다.

### Q4. 마이크로컨트롤러의 시리얼 통신은 어떤 방식으로 이루어지나?

- **답변:** 하나의 전송선(Wire)을 통해 데이터를 한 번에 1비트(Bit)씩 순차적으로 송수신하는 직렬 통신 방식으로 이루어집니다. 동기화를 위해 별도의 공통 클럭 신호선을 공유하는 **동기식 통신(I2C, SPI)**과, 클럭선 없이 미리 약속된 통신 속도(Baud Rate)와 스타트/스톱 비트를 기반으로 패킷 타이밍을 맞추는 **비동기식 통신(UART)**으로 분류되어 수행됩니다.

### Q5. 여러 센서를 연결할 때 I2C와 SPI 통신 중 어떤 방법이 좋을까?

- **답변:** 하드웨어 자원의 제약과 타깃 센서의 요구 속도에 따라 트레이드 오프(Trade-off) 관계가 성립하므로 상황에 맞게 선택해야 합니다.
  - **I2C 통신이 유리한 경우:** 마이크로컨트롤러의 핀 자원이 부족하고 센서의 데이터 전송 속도가 빠를 필요가 없을 때(예: 온도, 습도, 조도 센서 다중 연결) 좋습니다. 고유 주소 제어 방식 덕분에 장치가 아무리 늘어나도 단 2개의 선(SDA, SCL)만으로 버스 공유가 가능합니다.
  - **SPI 통신이 유리한 경우:** 데이터 전송량과 응답 속도가 극도로 중요할 때(예: 초고속 비전 센서 어레이, 대용량 데이터 로깅용 마이크로 SD 카드 구동) 좋습니다. 슬레이브 장치가 늘어날 때마다 독립적인 SS(Chip Select) 선이 추가되어 핀 소모는 심하지만, 주소 오버헤드가 없고 고속 전이중 통신이 가능하므로 실시간 고속 피드백 제어에 필수적입니다.

### Q6. 마이크로컨트롤러에서 타이머를 이용하는 이유는 무엇인가? 타이머가 제어 시스템에서 중요한 역할을 하는 이유는 무엇인가?

- **답변:** 메인 소프트웨어 루프(`void loop()`) 내의 연산 속도나 `delay()` 함수에 의존해 제어 주기를 잡으면 다른 코드의 실행 시간에 따라 샘플링 주기가 불규칙하게 흔들리는 지터(Jitter) 현상이 발생합니다. 피드백 제어계(특히 PID 제어)에서 주기가 흔들리면 수학적 미분 및 적분 오차가 기하급수적으로 누적되어 시스템이 발산(오작동)합니다. 하드웨어 타이머는 CPU 연산과 무관하게 공통 클럭을 카운트하여 정확한 데드라인마다 인터럽트를 트리거하므로, **일정한 제어 주기를 강제로 보장하여 제어 시스템을 안정화**시키는 중차대한 역할을 수행합니다.

### Q7. 타이머를 설정해 1초 간격으로 센서 데이터를 읽어오려면 어떤 절차를 따라야 할까?

- **답변:**
  1. **분주비(Prescaler) 계산 및 설정:** MCU의 메인 클럭(예: 16MHz)은 너무 빠르므로, 타이머 카운터가 1초 동안 포화되지 않고 셀 수 있도록 타이머 분주비를 설정(예: 1024 분주)하여 카운터 공급 주파수를 낮춥니다.
  2. **비교 일치 레지스터(OCR) 값 산출:** 설정한 분주 주파수를 기반으로 1초(1000ms)가 되는 정확한 타이머 카운터 도달 값(OCR)을 공식($\text{OCR} = \frac{\text{System Clock}}{\text{Prescaler} \times \text{Target Frequency}} - 1$)에 의해 구하여 레지스터에 대입합니다.
  3. **타이머 모드 및 인터럽트 인에이블:** 카운터 값이 OCR 값과 일치할 때 카운터를 0으로 초기화하는 CTC(Clear Timer on Compare Match) 모드를 활성화하고, 타이머 비교 일치 인터럽트 마스크를 턴온합니다.
  4. **ISR(인터럽트 서비스 루틴) 구현:** 1초마다 자동으로 트리거되는 해당 타이머의 고유 ISR 함수 내부에 센서 데이터를 읽어오는 전처리 코드를 배치하여 실행시킵니다.

### Q8. 아두이노를 마이크로컨트롤러가 아닌 마이크로보드라 부르는 이유는 무엇인가?

- **답변:** 마이크로컨트롤러(MCU)는 순수한 IC 반도체 칩 자체(예: ATmega328P)를 지칭하는 단어입니다. 아두이노는 이 칩이 독립적으로 정상 구동될 수 있도록 **전원 공급 가이드 레듈레이터, 클럭 공급용 크리스탈 발진기, PC 연동용 USB-to-Serial 변환 칩, 센서 결합용 입출력 헤더 소켓 핀 등을 하나의 인쇄회로기판(PCB) 위에 원보드로 일체화한 플랫폼**이기 때문에 마이크로보드(또는 개발 보드)라고 부르는 것이 올바릅니다.

### Q9. 아두이노의 디지털 핀, 아날로그 핀, PWM 핀 각각에 대해서 설명하라.

- **답변:** (Arduino Uno 기준)
  - **디지털 핀 (0~13번):** 상시 디지털 입출력이 가능한 핀으로, 0V와 5V의 이진 신호를 읽거나 출력합니다. 단, 0번과 1번은 PC 통신용 하드웨어 UART(RX/TX)와 맵핑되어 있습니다.
  - **아날로그 핀 (A0~A5번):** 내부의 10비트 고성능 ADC 통로와 연결된 핀으로, 0V부터 5V 사이의 연속적인 전압 변화를 0부터 1023 사이의 정수 데이터로 변환해 읽어옵니다.
  - **PWM 핀 (3, 5, 6, 9, 10, 11번):** 디지털 핀 중 번호 앞에 물결 표시(`~`)가 각인된 핀들입니다. 내부 타이머 카운터 레지스터와 연계되어 있어, 하드웨어적으로 디지털 신호의 듀티 사이클을 고속으로 가변 출력함으로써 아날로그 전압 제어 효과(모터 속도 제어 등)를 낼 수 있는 특수 핀입니다.
