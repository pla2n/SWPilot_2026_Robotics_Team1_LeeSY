# [과제 2-2] ROS2 개념 조사 및 실습 환경 구동 명세서

## 1. 로봇 운영체제(ROS)의 개념 및 역할

### 1.1. 로봇 운영체제(ROS)의 개념

- ROS(Robot Operating System)는 일반적인 컴퓨터 운영체제(Windows, Linux 등) 위에 얹어져 하드웨어 추상화, 저수준 디바이스 제어, 프로세스 간 메시지 패싱, 패키지 관리 기능 등을 제공하는 **로봇 전용 메타 운영체제(Meta-Operating System)**입니다.

### 1.2. 로봇을 제어하는 데 있어 ROS의 역할

- **하드웨어 추상화:** 복잡한 모터 드라이버나 센서 인터페이스를 규격화하여 개발자가 하드웨어의 미시적인 제어보다 알고리즘 구현에 집중할 수 있도록 돕습니다.
- **분산 프로세스 통신:** 로봇 시스템을 구성하는 다양한 소프트웨어 모듈(노드)들이 독립적으로 실행되면서 서로 유기적으로 데이터를 주고받을 수 있는 고성능 통신 인프라를 제공합니다.
- **생태계 및 도구 지원:** RViz(시각화), Gazebo(시뮬레이션) 등 검증된 로봇 개발 도구들을 즉시 연동할 수 있도록 지원합니다.

## 2. 운영체제 사용 유무에 따른 로봇 제어의 차이

| 구분                     | 운영체제(ROS) 기반 로봇                                                                           | 운영체제 미사용(Firmware/Bare-metal) 로봇                                                                  |
| :----------------------- | :------------------------------------------------------------------------------------------------ | :--------------------------------------------------------------------------------------------------------- |
| **제어 시스템 아키텍처** | **모듈형 분산 아키텍처**<br>센서 퓨전, SLAM, 내비게이션, 인공지능 연산 등 고차원 작업 처리에 적합 | **단일 루프/인터럽트 기반 제어**<br>MCU(마이크로컨트롤러) 레벨에서 모터 속도 제어 등 단순 반복 제어에 적합 |
| **하드웨어 변경 유연성** | **높음**<br>센서나 라이다를 교체해도 메시지 규격(Topic)만 맞추면 코드 재사용 가능                 | **낮음**<br>핀 맵(Pin map) 변경이나 하드웨어 사양 변경 시 메인 펌웨어 전체를 재수정해야 함                 |
| **통신 및 확장성**       | **탁월함**<br>네트워크를 통해 다수의 이기종 컴퓨터 및 외부 장치와 데이터 분산 처리 용이           | **제한적**<br>UART, SPI, I2C 등 온보드 수준의 저수준 통신 환경에 국한됨                                    |
| **실시간성 (Real-Time)** | 시스템 부하에 따라 미세한 지연(Latency) 가능성이 존재 (RTOS 커널 튜닝으로 보완)                   | 하드웨어 타이머와 구조적 특성 덕분에 **초정밀 실시간성(Hard Real-time)** 보장                              |

## 3. ROS2 Humble 패키지(Package)와 노드(Node)의 관계

### 3.1. 개념 정의

- **노드 (Node):** ROS2 시스템에서 특정 기능을 수행하는 **최소 단위의 실행 프로세스**입니다. (예: 카메라 영상을 읽는 노드, 바퀴 모터를 제어하는 노드 등)
- **패키지 (Package):** 하나 이상의 노드, 라이브러리, 설정 파일, 빌드 스크립트 등을 묶어놓은 **ROS2 소프트웨어의 가장 기본적인 배포 및 빌드 단위**입니다.

### 3.2. 구조적 관계

- 패키지와 노드는 **'포함 관계'**를 가집니다. 하나의 패키지 안에는 독립적으로 실행 가능한 여러 개의 노드가 포함될 수 있으며, 개발자는 관련성이 높은 노드들을 하나의 패키지로 묶어 관리합니다.

## 4. `ros2 run` 명령어 사용법 및 필수 환경 설정

### 4.1. `ros2 run` 명령어 문법

터미널 환경에서 특정 패키지 내에 포함된 노드를 탐색하고 실행할 때 사용합니다.

```bash
$ ros2 run <패키지_이름> <노드_실행파일_이름>
```
