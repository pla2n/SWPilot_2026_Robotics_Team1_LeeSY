# [과제 2-9] ROS2 서브스크라이버(Subscriber) 구현 및 로봇 위치 데이터 분석

## 1. /turtle1/pose 토픽 및 메시지 구조의 심층 조사

`turtlesim_node`가 실행될 때 시스템 내부에서 로봇의 실시간 상태를 외부로 퍼블리시하는 핵심 토픽이 바로 `/turtle1/pose`입니다. 이 토픽은 단순히 거북이의 위치를 화면에 보여주는 것에 그치지 않고, 자율주행 로봇 공학에서 가장 중요한 **피드백 제어(Feedback Control)**의 기초가 되는 데드 레코닝(Dead Reckoning) 데이터를 담고 있습니다.

### 1.1 `ros2 interface show` 명령의 매커니즘

ROS2 통신 시스템은 노드 간 데이터를 주고받을 때 엄격하게 약속된 데이터 구조인 **인터페이스(Interface)**를 사용합니다. `ros2 interface show` 명령은 특정 메시지 파일(이 패키지에서는 `.msg` 파일)의 원본 정의를 파싱하여, 해당 메시지가 어떤 원시 자료형(Primitive Data Type)과 변수명으로 구성되어 있는지 터미널 상에 명확하게 디스플레이해 줍니다.

개발자는 이 명령을 통해 소스 코드 내부에서 객체의 속성(Attribute)에 오타 없이 정확하게 접근할 수 있는 명세서를 확보하게 됩니다.

### 1.2 `/turtle1/pose` 메시지 유형 (`turtlesim/msg/Pose`) 구조 분석

터미널에서 `ros2 topic info /turtle1/pose`를 실행하면 해당 토픽의 타입이 `turtlesim/msg/Pose`임을 확인할 수 있습니다. 이를 `ros2 interface show turtlesim/msg/Pose` 명령어로 조회하면 아래와 같이 5개의 `float32` 자료형 데이터로 구성되어 있음을 볼 수 있습니다.

- **`float32 x`**
  - **의미:** turtlesim 시뮬레이터 격자 지도(Grid Map) 좌표계 기준 거북이의 현재 X축 위치를 나타냅니다.
  - **해석:** 화면 좌측 끝이 0에 가까우며 우측 끝으로 갈수록 값이 커집니다. (기본 화면 크기는 대략 0.0부터 11.0 사이의 범위를 가집니다.)
- **`float32 y`**
  - **의미:** 시뮬레이터 격자 지도 좌표계 기준 거북이의 현재 Y축 위치를 나타냅니다.
  - **해석:** 화면 하단 끝이 0에 가까우며 상단 끝으로 갈수록 값이 커집니다. X축과 Y축이 만나는 대각선 교차점이 거북이의 2차원 절대 위치가 됩니다.
- **`float32 theta`**
  - **의미:** 거북이가 바라보고 있는 정면(Heading)의 회전 각도를 나타냅니다.
  - **해석:** ROS2 기구학 스탠다드에 따라 **라디안(Radian)** 단위를 사용하며, 범위는 $-\pi$ ~ $+\pi$ (약 -3.14159 ~ +3.14159) 사이로 표현됩니다. 거북이가 우측(X축 양의 방향)을 바라볼 때가 `0`이며, 반시계 방향으로 회전할수록 값이 커지고 시계 방향으로 회전할수록 음수 값으로 떨어집니다.
- **`float32 linear_velocity`**
  - **의미:** 거북이의 현재 선속도(직진 이동 속도)를 나타냅니다.
  - **해석:** 로봇이 전진하거나 후진할 때의 순간 속도이며, 단위는 초당 이동 거리($m/s$)입니다. 가만히 멈춰 서 있으면 `0`이 됩니다.
- **`float32 angular_velocity`**
  - **의미:** 거북이의 현재 각속도(회전 속도)를 나타냅니다.
  - **해석:** 로봇이 제자리에서 꺾거나 회전할 때의 순간 회전 속도이며, 단위는 초당 회전 라디안($rad/s$)입니다. 직선 운동만 할 때는 `0`이 됩니다.

이 메시지는 파이썬 소스 코드 내부로 임포트될 때 `turtlesim.msg` 모듈의 `Pose` 클래스로 매핑되며, 콜백 함수 내부에서 `msg.x`, `msg.y`, `msg.theta` 처럼 점(dot) 연산자를 통해 인스턴스 변수 형태로 다루어집니다.

## 2. 파이썬 서브스크라이버(Subscriber) 노드 생성 원리 및 구조

구독자를 생성하기 위해서는 `rclpy.node.Node` 클래스가 가진 상속 메서드인 `create_subscription()`을 활용합니다.

### 2.1 퍼블리셔와의 구조적 차이점 (타이머를 쓰지 않는 이유)

퍼블리셔(Publisher)는 데이터를 언제 보낼지 능동적으로 결정해야 하므로 `create_timer()` 같은 타이머 스케줄러를 붙여 주기적으로 콜백을 발생시켜야 합니다.

반면, 서브스크라이버(Subscriber)는 내부 타이머를 일절 사용하지 않고 **'이벤트 기반(Event-driven) 매커니즘'**으로 동작합니다.

[Image of ROS2 Event-driven callback queue execution pipeline]

1. 토픽 채널을 모니터링하고 있다가 퍼블리셔(`turtlesim_node`)가 데이터를 통신망에 게시하는 순간이 하나의 **'이벤트(Event)'**가 됩니다.
2. ROS2의 미들웨어(DDS)가 이 이벤트 데이터를 수신하여 노드의 콜백 큐(Callback Queue)에 밀어 넣습니다.
3. 메인 스레드에서 무한 루프를 돌며 대기 중인 `rclpy.spin()` 엔진이 큐에 들어온 메시지를 포착하는 즉시, 우리가 매핑해 둔 구독자 콜백 함수(`get_turtle_pose`)를 자동으로 깨워 실행시킵니다.
4. 따라서 구독자는 주기를 고민할 필요 없이, **데이터가 들어오는 시점(Turtlesim의 경우 약 60Hz 주기로 발생)에만 수동적이고 효율적으로 반응**하게 설계됩니다.

### 2.2 데이터 변경 시에만 로그를 남기는 조건 제어 매커니즘

`turtlesim_node`는 로봇이 움직이지 않고 가만히 멈춰 서 있는 상태에서도 현재 상태 보존을 위해 똑같은 좌표 데이터(예: X=5.54, Y=5.54, Theta=0.0)를 초당 60번씩 무한히 퍼블리시합니다. 만약 조건 검사 없이 콜백이 올 때마다 무조건 화면에 텍스트를 출력하게 만들면 다음과 같은 치명적인 문제가 발생합니다.

- 터미널 창에 똑같은 메시지가 폭포수처럼 쏟아져 인간 개발자가 실시간 로그를 식별하는 것이 불가능해집니다.
- 콘솔에 문자열을 출력하는 I/O 작업은 컴퓨터 자원(CPU)을 엄청나게 잡아먹으므로, 가만히 있는 로봇 때문에 불필요한 시스템 과부하가 걸립니다.

이를 해결하기 위해 본 과제에서는 **속도 데이터를 제외한 순수 위치 정보(x, y, theta)에 변화가 생겼을 때만 조건부로 로깅**하는 필터링 메커니즘을 클래스 구조 내부에 구현했습니다.

- **상태 보존 변수 선언:** 노드 클래스가 초기화되는 `__init__` 시점에 직전 루프의 데이터를 기억해 둘 멤버 변수(`self.last_x`, `self.last_y`, `self.last_theta`)를 생성하고 `None`으로 세팅합니다.
- **차단 및 비교 조건문:** 콜백 함수가 트리거되면 `if` 문을 통해 `(self.last_x != msg.x or self.last_y != msg.y or self.last_theta != msg.theta)` 검사를 수행합니다.
- **최초 실행 및 변동 시 처리:** 처음 노드가 켜졌을 때는 변수들이 `None`이므로 조건문이 `True`가 되어 첫 위치를 강제로 한 번 출력합니다. 이후 거북이가 움직여서 셋 중 하나라도 이전 값과 달라지면 조건문이 성립되어 `self.get_logger().info()`를 통해 좌표 로그를 남깁니다.
- **값의 갱신(Update):** 로그를 출력한 직후, 다음 번 들어올 메시지와 비교할 수 있도록 `self.last_x = msg.x` 처럼 현재 수신한 최신 값을 이전 값 변수에 대입하여 덮어씁니다. 만약 거북이가 멈춰 있어서 값이 똑같다면 `if` 문 내부를 통과하지 못하므로 터미널은 완벽하게 정적 상태를 유지하게 됩니다.

## 3. 의존성(Dependency) 설정과 빌드 시스템 구조적 연계

### 3.1 `package.xml` 메타데이터 의존성 선언의 본질

`package.xml`은 이 패키지가 구동되기 위해 리눅스 운영체제 및 ROS2 환경에 설치되어 있어야 하는 의존성 패키지 리스트를 선언하는 명세서입니다.

```xml
  <depend>rclpy</depend>
  <depend>geometry_msgs</depend>
  <depend>turtlesim</depend>
```
