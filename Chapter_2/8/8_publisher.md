# [과제 2-8] ROS2 퍼블리셔(Publisher) 구현 및 거북이 원 그리기 제어

---

## 1. 노드 간 퍼블리시(Publish)와 서브스크라이브(Subscribe) 관계 분석

이전 실습에서 관찰한 `turtlesim_node`(거북이 시뮬레이터)와 `turtle_teleop_key`(키보드 조종기)의 통신 구조는 ROS2의 핵심 통신 아키텍처인 **게시/구독(Publish-Subscribe) 모델**을 명확하게 보여줍니다.

- **게시자 (Publisher):** 데이터를 생성하여 네트워크 상에 뿌리는 역할을 합니다. `turtle_teleop_key` 노드는 사용자가 누른 키보드 방향키 입력을 속도 데이터로 변환한 뒤, `/turtle1/cmd_vel`이라는 토픽(Topic)에 쉴 새 없이 **게시(Publish)**합니다.
- **구독자 (Subscriber):** 필요한 데이터를 수신하기 위해 특정 토픽의 주파수를 맞추고 대기하는 역할입니다. `turtlesim_node`는 `/turtle1/cmd_vel` 토픽을 **구독(Subscribe)**하고 있다가, 새로운 속도 데이터가 들어오면 이를 읽어들여 화면 속 거북이의 위치를 갱신(이동)시킵니다.

이 구조의 가장 큰 장점은 **비동기성과 익명성**입니다. 데이터를 보내는 쪽과 받는 쪽이 서로 누군지 몰라도, 오직 **'토픽 이름'**만 일치하면 데이터를 완벽하게 주고받을 수 있습니다.

## 2. 통신 메시지 유형(Type) 및 구성 요소의 의미

두 노드가 통신하기 위해서는 서로 주고받는 데이터의 '모양(규격)'이 일치해야 합니다. `/turtle1/cmd_vel` 토픽에서 사용하는 메시지 규격은 **`geometry_msgs/msg/Twist`** 입니다.

### 2.1 Twist 메시지의 내부 구조와 의미

`Twist` 메시지는 3차원 공간에서 물체(강체)의 이동 속도를 표현하기 위해 총 6개의 실수(Float) 데이터로 이루어져 있습니다.

1. **`linear` (선속도 벡터): 직선 방향으로 나아가는 속도**
   - `x`: 로봇의 앞/뒤 전진 및 후진 속도 (거북이가 바라보는 정면 방향)
   - `y`: 로봇의 좌/우 '게걸음' 속도 (일반적인 바퀴 로봇은 불가능하므로 주로 0)
   - `z`: 로봇의 상/하 수직 비행 속도 (드론 제어 시 사용)

2. **`angular` (각속도 벡터): 축을 중심으로 회전하는 속도**
   - `x`: Roll (앞뒤 축을 기준으로 비행기 날개가 기울듯 회전)
   - `y`: Pitch (좌우 축을 기준으로 자동차가 언덕을 오르듯 앞뒤가 들림)
   - `z`: Yaw (위아래 축을 기준으로 팽이처럼 좌/우로 방향을 틂)

**결론:** 2차원 평면을 돌아다니는 거북이를 제어할 때는 앞으로 가는 `linear.x`와 좌우로 방향을 꺾는 `angular.z` 단 두 개의 값만 사용하게 됩니다.

## 3. 파이썬을 활용한 퍼블리셔(Publisher) 노드 작성 원리

사용자가 직접 조종하지 않고 거북이가 스스로 원을 그리게 만들려면, 파이썬을 이용해 `turtle_teleop_key`를 대신할 퍼블리셔 노드를 만들어야 합니다.

### 3.1 퍼블리셔 생성 문법: `create_publisher()`

파이썬 `Node` 클래스 내부에서 아래와 같이 퍼블리셔 객체를 생성합니다.

> `self.create_publisher(Twist, "/turtle1/cmd_vel", 10)`

- **첫 번째 인자 (`Twist`):** 발행할 메시지의 데이터 규격
- **두 번째 인자 (`"/turtle1/cmd_vel"`):** 데이터를 던질 목적지 토픽의 이름
- **세 번째 인자 (`10`):** 큐 사이즈(Queue Size). 네트워크 지연 시 데이터를 임시 보관할 버퍼의 크기로, 통신의 안정성을 보장합니다.

### 3.2 원을 그리는 원리 (제어 로직)

타이머를 생성하여 0.1초마다 `Twist` 메시지를 발행하도록 구성합니다. 이때 메시지 객체의 `linear.x = 1.0` (초당 1만큼 전진), `angular.z = 1.0` (초당 1라디안 회전)으로 설정합니다. 로봇이 앞으로 나아가면서 동시에 일정한 속도로 방향을 틀기 때문에, 결과적으로 **부드러운 원 궤적**을 그리며 이동하게 됩니다.

---

## 4. 의존성(Dependency) 추가 (`package.xml`)

우리가 작성한 노드는 `geometry_msgs` 패키지의 `Twist` 클래스를 사용하고, `turtlesim` 노드와 상호작용합니다. 따라서 빌드 시스템이 이를 인지할 수 있도록 `package.xml` 파일에 아래 의존성을 추가해야 합니다.

```xml
  <depend>geometry_msgs</depend>
  <depend>turtlesim</depend>
```
