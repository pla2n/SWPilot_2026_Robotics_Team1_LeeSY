# [과제 2-12] ROS2 Launch File 작성 및 다중 노드 자동화 실행

---

## 1. ROS2 Launch File의 개념과 도입 목적

이전 실습까지는 로봇을 제어하기 위해 터미널을 여러 개 열고, 각 터미널마다 `ros2 run` 명령어를 일일이 입력하여 시뮬레이터(`turtlesim_node`)와 제어 노드(`turtle_move_control`)를 개별적으로 실행해야 했습니다.

하지만 실제 로봇 환경(예: 자율주행 자동차, 드론)에서는 라이다(LiDAR) 센서 노드, 카메라 노드, 모터 제어 노드, 위치 추정 노드 등 수십에서 수백 개의 노드가 동시에 구동되어야 합니다. 이를 사람이 터미널에서 하나씩 켜는 것은 불가능에 가깝습니다.

**Launch File(런치 파일)**은 이렇게 복잡한 로봇 시스템에서 **여러 개의 노드를 단 한 번의 명령어로 동시에 실행시키고, 각 노드에 필요한 파라미터(설정값)나 네임스페이스 등을 일괄적으로 관리 및 자동화하기 위한 목적**으로 사용되는 강력한 스크립트 파일입니다.

---

## 2. 파이썬(Python)을 활용한 Launch File 작성법

ROS1에서는 XML 형식을 주로 사용했지만, ROS2에서는 파이썬(Python)을 이용해 Launch File을 작성하는 것이 표준입니다. 파이썬을 사용함으로써 단순한 실행을 넘어 조건문, 반복문 등을 활용한 동적인 실행 제어가 가능해졌습니다.

### 2.1 핵심 모듈 임포트

작성을 위해서는 ROS2의 런치 시스템에서 제공하는 두 가지 핵심 라이브러리를 임포트해야 합니다.

- `from launch import LaunchDescription`: 실행할 작업들의 전체 목록(명세서)을 담는 그릇입니다.
- `from launch_ros.actions import Node`: ROS2 노드를 실행하기 위한 액션 정의 클래스입니다.

### 2.2 `generate_launch_description()` 함수 정의

ROS2 런치 시스템은 파이썬 파일을 읽을 때 무조건 **`generate_launch_description()`** 이라는 이름의 함수를 찾아서 실행하도록 약속되어 있습니다. 따라서 이 함수를 정의하고, 내부에서 `LaunchDescription` 객체를 생성하여 반환(Return)하도록 구조를 잡아야 합니다.

### 2.3 `Node` 액션 파라미터 구성

`LaunchDescription` 배열 안에는 구동할 노드들의 정보를 `Node(...)` 형태로 나열합니다.

- `package`: 실행할 노드가 포함된 패키지 이름 (예: `turtlesim`, `my_robot_controller`)
- `executable`: 실제 실행할 바이너리 또는 스크립트의 이름 (`setup.py`에 등록한 이름)
- `name`: 실행될 때 ROS2 네트워크상에 부여할 노드의 별명
- `output`: `screen`으로 설정하면 노드에서 발생하는 로그나 출력을 터미널 화면에 바로 띄워줍니다.

## 3. Launch File 인식을 위한 `setup.py` 구조 수정

파이썬 코드(`.py`)로 노드를 짰을 때 `setup.py`의 `console_scripts`에 진입점을 등록했던 것처럼, Launch File 역시 빌드 시스템인 `colcon`이 이를 인식하고 올바른 경로로 복사할 수 있도록 `setup.py`를 수정해야 합니다.

### 3.1 `data_files` 필드의 역할

`setup.py` 내부의 `data_files` 배열은 패키지가 설치(Install)될 때 바이너리 실행 파일이 아닌 **부수적인 리소스 파일들(Launch File, 파라미터 파일, 환경 설정 파일 등)을 시스템의 지정된 공유 폴더(`share/패키지명/`)로 복사해 주는 역할**을 담당합니다.

### 3.2 경로 매핑 설정

런치 파일을 등록하려면 파일 및 경로 탐색을 위한 `os`와 `glob` 모듈을 임포트한 뒤, `data_files` 리스트 안에 아래와 같은 튜플(Tuple) 구조를 추가해야 합니다.

> `(os.path.join('share', package_name, 'launch'), glob('launch/*.launch.py'))`

이 구문의 의미는 **"현재 패키지 폴더 안의 `launch/` 디렉토리에 있는 `.launch.py`로 끝나는 모든 파일을, 빌드 후 `share/패키지명/launch/` 경로로 복사해라"**라는 뜻입니다. 이 과정을 거쳐야만 ROS2 명령어가 런치 파일을 정상적으로 추적하여 실행할 수 있습니다.

---

## 4. `ros2 launch` 명령어 사용법

단일 노드를 실행하던 `ros2 run`과 달리, 런치 파일을 실행할 때는 `ros2 launch` 명령어를 사용합니다.

> **문법:** `ros2 launch <패키지_이름> <런치_파일_이름>`
> **예시:** `ros2 launch my_robot_controller my_robot_launch.launch.py`

이 명령어를 입력하면 런치 파일에 기록된 명세서(LaunchDescription)를 시스템이 읽어 들여, 터미널 하나에서 `turtlesim_node`와 우리가 만든 `turtle_move_control` 제어 노드를 동시에 병렬 스레드로 띄우고 통신망을 자동으로 구축합니다.

## 5. [부록] 평가 가이드라인 기반 심층 Q&A 검증

### Q1. ROS2의 Launch File은 어떤 목적으로 사용되는 파일인가?

- **답변:** 수많은 노드로 이루어진 복잡한 로봇 소프트웨어 시스템에서, 여러 개의 노드를 터미널 하나에서 한 번의 명령으로 동시에 실행시키고 관리하기 위한 목적으로 사용되는 자동화 스크립트 파일입니다. 노드의 네임스페이스나 파라미터를 중앙에서 일괄 제어할 때도 필수적으로 사용됩니다.

### Q2. ROS2의 Launch File의 작성법에 대해서 설명해보자.

- **답변:** 파이썬을 기반으로 작성하며, 핵심 규약인 `generate_launch_description()` 함수를 선언해야 합니다. 이 함수 내부에서 `launch_ros.actions.Node` 클래스를 이용해 실행할 대상 노드들의 정보(패키지명, 실행 파일명 등)를 정의하고, 이 노드들을 `LaunchDescription` 리스트 객체에 담아 최종적으로 반환(return)하는 절차로 작성됩니다.

### Q3. `setup.py` 파일의 `setup.data_files`는 ROS2의 패키지 설치에서 어떤 역할을 하는지 추정해보자.

- **답변:** 파이썬 실행 코드 자체가 아니라, 패키지 구동에 필요한 메타 데이터나 설정 파일(Launch File, 맵 파일, URDF 모델 등)의 설치 목록과 목적지 경로를 지정하는 역할을 합니다. 즉, 소스 코드 폴더(`src`)에 있는 리소스 파일들을 빌드 결과물이 위치하는 `install/share/패키지명/` 경로로 알맞게 복사(배포)하여, ROS2 시스템이 런타임에 해당 리소스들을 찾아 읽을 수 있도록 만들어 주는 중요한 연결 고리 역할을 합니다.
