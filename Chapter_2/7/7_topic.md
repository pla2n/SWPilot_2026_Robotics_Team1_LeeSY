# [과제 2-7] ROS2 토픽(Topic) 통신 매커니즘 분석 및 실습

---

## 1. ROS2 토픽(Topic)의 개념

- **정의:** ROS2 노드(Node)들이 서로 데이터를 주고받기 위해 사용하는 **단방향 비동기 통신 버스(채널)**이다.
- **특징 (Publish-Subscribe 모델):** [Image of ROS2 publish subscribe topic architecture] 데이터를 보내는 노드를 **게시자(Publisher)**, 데이터를 받는 노드를 **구독자(Subscriber)**라고 부른다. 이들은 서로의 존재를 직접 알 필요 없이, 오직 '토픽의 이름'을 매개체로 익명 통신을 수행한다.

---

## 2. `rqt_graph`를 통한 통신망 시각적 분석

`talker`와 `listener` 노드를 실행한 상태에서 `rqt_graph`를 실행하고 옵션을 조작하여 시스템 토폴로지를 분석하였다.

- **Nodes/Topics (active) 모드:** 순수 노드 객체뿐만 아니라, 그들 사이에 매개체 역할을 하는 네모난 **'토픽(Topic)' 블록(`/chatter`)**이 함께 시각화된다.
- **연결선의 방향성:** 화살표가 `talker` 노드에서 출발하여 `/chatter` 토픽을 거쳐 `listener` 노드로 향한다. 이는 데이터가 게시자에서 토픽으로, 토픽에서 다시 구독자로 흘러가는 **정보의 명확한 전달 방향(단방향 통신)**을 의미한다.

---

## 3. `ros2 topic` 핵심 명령어 분석

새 터미널을 열고 `/chatter` 토픽에 대한 정보를 분석하였다.

### 3.1 `ros2 topic list`

- **실행 결과:** 시스템에 현재 개설되어 활성화된 모든 토픽의 이름 목록(`/chatter`, `/rosout`, `/parameter_events` 등)을 출력한다.

### 3.2 `ros2 topic info /chatter`

- **실행 결과:** ```text
  Type: std_msgs/msg/String
  Publisher count: 1
  Subscription count: 1
  ```

  ```
- **의미:** `/chatter` 토픽을 통해 오가는 데이터의 자료형(Type)이 문자열(`String`)이며, 현재 이 토픽에 데이터를 뿌리는 노드(Pub)가 1개, 듣고 있는 노드(Sub)가 1개 있음을 알려준다.

### 3.3 `ros2 topic echo /chatter`

- **실행 결과:** `data: 'Hello World: 133'` ... (실시간 출력)
- **의미:** 터미널 자체가 임시 구독자(Subscriber)가 되어, `/chatter` 토픽에 흘러다니는 실제 데이터 값을 가로채어 화면에 실시간으로 감청(Echo)해 보여준다.

## 4. ROS2의 기본 메시지 유형 (Message Types)

노드 간 통신이 이루어지려면 송신자와 수신자가 동일한 데이터 규격을 사용해야 한다. 이를 **메시지(Message)**라 부른다.

- **`std_msgs` (Standard Messages):** 프로그래밍 언어의 기본 자료형들이다. (예: `String`(문자열), `Int32`(정수), `Float64`(실수), `Bool`(참/거짓) 등)
- **`geometry_msgs`:** 로봇의 물리적 이동이나 기하학적 정보를 담기 위한 복합 자료형이다. (예: `Twist`(선속도/각속도 벡터), `Pose`(위치 및 자세), `Point` 등)

## 5. 다대다(N:M) 통신 메커니즘 검증 실험

터미널을 여러 개 열어 노드 개수를 인위적으로 조절하며 `ros2 topic info /chatter`의 결과를 관찰하였다.

| 상황                     | Publisher count | Subscription count | 의미 및 분석                                                                                                                                      |
| :----------------------- | :-------------- | :----------------- | :------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Talker 1, Listener 2** | 1               | 2                  | 1개의 센서(또는 제어기) 데이터를 2개의 서로 다른 노드가 동시에 가져다 쓰는 **1:N 통신 브로드캐스팅**이 완벽히 지원됨을 의미한다.                  |
| **Talker 2, Listener 1** | 2               | 1                  | 2개의 서로 다른 노드가 동일한 토픽에 데이터를 발행해도, 구독자는 이를 병합하여 모두 수신할 수 있는 **N:1 통신**이 성립함을 의미한다.              |
| **Talker 2, Listener 2** | 2               | 2                  | **완전한 N:M (다대다) 분산 통신망**을 형성한다. 이는 ROS2가 중앙 서버 없이 DDS(데이터 분산 서비스)를 통해 각 노드를 유연하게 결합시킴을 증명한다. |

## 6. Turtlesim 다중 노드 제어 실습 및 분석

거북이 시뮬레이터(`turtlesim_node`)를 2개 띄우고, 키보드 조종 노드(`turtle_teleop_key`) 1개를 실행하여 동작을 관찰하였다.

### 6.1 관찰 결과 (이미지: `turtlesim_screen_2.png`)

- 키보드 방향키를 한 번만 눌러도 **독립적으로 띄워진 2개의 화면 속 거북이가 완전히 동일한 궤적과 속도로 동시에 움직이는(동기화)** 놀라운 현상이 발생한다.

### 6.2 `/turtle1/cmd_vel` 토픽 정보 분석

- **Publisher / Subscription 수:** Publisher count: 1 (키보드 노드), Subscription count: 2 (시뮬레이터 노드 2개)
- **토픽 형태(Type):** `geometry_msgs/msg/Twist`
- **토픽 내용 (echo 캡처):**
  ```text
  linear:
    x: 2.0
    y: 0.0
    z: 0.0
  angular:
    x: 0.0
    y: 0.0
    z: 0.0
  ```
- **종합 정리:** 키보드 노드(Pub)가 전진 명령(`linear x: 2.0`)이라는 `Twist` 형태의 메시지를 `/turtle1/cmd_vel` 토픽에 한 번 발행하면, 이 토픽을 구독하고 있던 2개의 시뮬레이터 노드(Sub)가 동시에 데이터를 낚아채어 각자의 화면에 거북이를 전진시키는 원리이다.

## 7. [부록] 평가 가이드 기반 문제 이해도 검증

### Q1. `rqt_graph`에서 노드, 토픽 간 연결선의 화살표 방향은 어떤 의미를 지니는가?

- **답변:** 데이터(메시지)가 실제로 흘러가는 **정보의 전달 방향**을 의미합니다. 게시자에서 토픽으로, 토픽에서 구독자로 향합니다.

### Q2. 토픽, 게시자, 구독자의 관계를 설명해보자.

- **답변:** **게시자(Publisher)**는 데이터를 생성해 공용 게시판인 **토픽(Topic)**에 올려두고, 해당 데이터를 필요로 하는 **구독자(Subscriber)**는 게시판(토픽)에 올라온 데이터를 스스로 가져가서 사용하는 유연한 결합 메커니즘입니다.

### Q3. `/chatter` 토픽과 `/turtle1/cmd_vel` 토픽에 저장되는 정보는 어떻게 다른지 설명해보자.

- **답변:** `/chatter` 토픽은 `std_msgs/msg/String` 유형으로 단순히 한 줄의 **단일 텍스트 문자열** 정보를 담습니다. 반면, `/turtle1/cmd_vel` 토픽은 `geometry_msgs/msg/Twist` 유형으로 로봇의 3차원 선속도(linear x,y,z)와 각속도(angular x,y,z)를 나타내는 **총 6개의 실수(Float) 데이터 묶음**을 복합적으로 담고 있습니다.

### Q4. 하나의 `teleop` 노드와 두 개의 `turtlesim` 노드 실행 시 정보 교환 방식을 설명해보자.

- **답변:** 1개의 조종 노드가 키보드 입력을 받아 `/turtle1/cmd_vel`이라는 토픽 채널에 이동 명령(Twist)을 뿌립니다(Publish). 그러면 이 채널의 주파수를 맞추고 있는(Subscribe) 2개의 시뮬레이터 노드가 해당 명령을 동시에 가져가 각자의 화면에 그려내는 **1:N 브로드캐스팅 방식**으로 통신합니다.

### Q5. ROS2 노드 간 기본 통신 메커니즘의 결론을 요약해보자.

- **답변:** 정해진 **데이터 유형(Type)**과 **토픽 이름(Name)**이라는 규약만 맞으면, 동일한 이름의 토픽에 하나 이상의 게시자가 자유롭게 정보를 뿌릴 수 있고 하나 이상의 구독자가 제약 없이 정보를 가져갈 수 있는 **유연한 다대다(N:M) 분산 통신망**이 기본 통신 메커니즘입니다.

### Q6. 별다른 설정 없이 실행된 구독자는 실행 이전에 게시된 정보를 가져가는가?

- **답변:** **가져가지 않습니다.** 기본적으로 토픽은 실시간 스트리밍 방식(Best-effort)이므로, 구독자가 켜지기 전에 이미 지나가버린 과거의 메시지는 유실됩니다. (이를 보완하기 위해 QoS 설정이나 Service/
