# [과제 2-3] ROS2 노드(Node) 개념 조사 및 rqt_graph 통신 시각화 실습

## 1. ROS2 노드(Node)의 개념

- **정의:** ROS2 시스템에서 단일 목적의 연산을 수행하는 **가장 기본적인 실행 단위(객체)**입니다.
- **특징:** 로봇을 제어하는 모든 하드웨어 요소(센서, 모터 등)나 소프트웨어 알고리즘(자율주행, 화면 출력 등)은 각각 하나의 독립된 노드로 매핑되어 실행됩니다.
- **장점:** 프로그램이 하나의 거대한 덩어리가 아닌 여러 노드로 분산되어 있기 때문에, 특정 노드에 에러가 발생해도 전체 시스템이 멈추지 않아 높은 안정성(Fault Tolerance)을 가지며 코드의 재사용성이 뛰어납니다.

---

## 2. `ros2 node` 명령어 분석

### 2.1. `ros2 node list`

- 현재 ROS2 네트워크 상에서 **활성화되어 실행 중인 모든 노드의 이름 목록**을 출력합니다.
- 데모 실행 후 터미널에 입력하면 `/talker`와 `/listener` (또는 `/turtlesim` 등)가 정상적으로 출력되며, 이는 `rqt_graph` 화면에 나타난 타원형 노드 객체들과 정확히 일치합니다.

### 2.2. `ros2 node info <노드_이름>`

- 특정 노드의 **상세한 통신 정보**를 조회합니다.
- 해당 노드가 어떤 데이터(Topic)를 발행(Publisher)하고 구독(Subscriber)하는지, 그리고 어떤 서비스(Service)와 액션(Action)을 활성화해 두었는지 구체적인 인터페이스 구조를 확인할 수 있습니다.

---

## 3. Turtlesim 패키지 실습 및 노드 관계 분석

### 3.1. Turtlesim 구동

두 개의 터미널을 열고 각각 다음 명령을 실행하여 거북이 시뮬레이터를 가동했습니다.

1. **시뮬레이터 화면 노드:** `ros2 run turtlesim turtlesim_node`
2. **키보드 제어 노드:** `ros2 run turtlesim turtle_teleop_key`

### 3.2. 두 노드 간의 통신 관계

- **제어 흐름:** 사용자가 터미널에서 방향키를 누르면 `turtle_teleop_key` 노드가 이를 인식하여 이동 속도와 방향 명령(데이터)을 발행합니다. 그러면 화면을 띄우고 있는 `turtlesim_node`가 이 데이터를 구독하여 화면 속 거북이의 위치를 업데이트합니다.
- 즉, 키보드 노드는 **명령 발행자(Publisher)**, 시뮬레이터 노드는 **명령 구독자(Subscriber)**의 관계를 형성하며 상호작용합니다.

## 4. [부록] 평가 가이드 기반 이해도 검증 답변

### Q1. ROS2의 노드에 대해서 설명해보자.

- **답변:** 로봇을 구성하는 수많은 부품과 기능들(카메라, 바퀴 제어, 라이다 센서 등)을 각각 제어하기 위해 쪼개어 놓은 **'단일 목적의 독립적인 실행 프로세스(객체)'**입니다. ROS2 환경에서 제어 가능한 모든 요소는 최종적으로 이 노드 단위로 동작하게 됩니다.

### Q2. 'Nodes only' 모드에서 보여주는 rqt_graph 화면의 그래프에서 어떤 것들을 확인할 수 있는지 설명해보자.

- **답변:** 타원형으로 표시된 **현재 동작 중인 노드들의 직관적인 목록**을 확인할 수 있으며, 화살표 모양의 선을 통해 데이터가 어느 노드에서 출발하여 어느 노드로 흘러 들어가는지 **통신의 방향성과 관계 구조**를 한눈에 파악할 수 있습니다.

### Q3. 노드의 정보를 확인하는 방법을 설명해보자.

- **답변:** 터미널 창에 `ros2 node info /<확인할_노드_이름>` 명령어를 입력하면 됩니다. 이를 통해 해당 노드가 주고받는 메시지(Topic)의 이름과 자료형 등 상세한 스펙을 텍스트 형태로 조회할 수 있습니다.

### Q4. Turtlesim 패키지의 두 노드를 실행한 상태에서 rqt_graph를 보면 두 노드 사이에 세 개의 선이 방향에 따라 연결되어 있다. 여기서 추정 가능한 것은 무엇인가?

- **답변:** 첫째, 화살표의 방향을 통해 데이터를 주는 쪽(Publisher)과 받는 쪽(Subscriber)의 **단방향 통신 흐름**이 존재함을 알 수 있습니다. 둘째, 선이 세 개라는 것은 두 노드 사이에서 단순히 하나의 데이터만 주고받는 것이 아니라, 위치 정보, 속도 제어 등 **서로 다른 목적을 가진 여러 개의 독립적인 통신 채널(Topic)이 동시에 작동하고 있음**을 유추할 수 있습니다.
