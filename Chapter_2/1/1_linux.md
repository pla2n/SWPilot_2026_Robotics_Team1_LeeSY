# [과제 2-1] ROS2 실습을 위한 우분투 리눅스 환경 구성 및 기초 명령어 명세서

## 1. 우분투 리눅스 실습 환경 구성

- **운영체제:** Ubuntu 22.04 LTS
- **설치 방식:** 가상 머신(VM) 환경 구축 완료
- **추가 설치 프로그램:** Chrome 웹 브라우저, Visual Studio Code 설치 완료

---

## 2. 리눅스 운영체제 기초 명령어 및 개념 조사

### 2.1. 터미널(Terminal) 실행 방법

- **GUI 활용:** 우분투 바탕화면 하단의 'Show Applications' 아이콘을 클릭한 후 `terminal`을 검색하여 실행합니다.
- **단축키 활용:** 리눅스 표준 단축키인 **`Ctrl + Alt + T`**를 누르면 즉시 bash 셸이 내장된 터미널이 실행됩니다.

### 2.2. 파일 및 디렉토리 관리 명령어

- **`pwd`**: 현재 위치한 작업 디렉토리의 절대 경로를 출력합니다.
- **`cd [경로]`**: 지정한 작업 디렉토리로 이동합니다. (예: `cd ~/study`)
- **`mkdir [이름]`**: 새로운 디렉토리를 생성합니다. (예: `sudo mkdir /test`)
- **`rmdir [이름]`**: 비어있는 디렉토리를 삭제합니다.
- **`ls`**: 현재 디렉토리 내의 파일과 폴더 목록을 출력합니다. (`-al` 옵션 시 권한/숨김 파일 포함)
- **`rm [이름]`**: 파일이나 디렉토리를 삭제합니다. (`-rf` 옵션 시 디렉토리 내부까지 강제 삭제)
- **`cp [원본] [사본]`**: 파일이나 디렉토리를 복사합니다.
- **`mv [원본] [목적지]`**: 파일/디렉토리를 이동하거나 이름을 변경합니다.

### 2.3. 파일 내용 확인 및 편집

- **내용 확인 (`cat`):** `cat /test/result.txt` 명령어를 사용하여 텍스트 내용을 터미널 화면에 즉시 출력합니다.
- **내용 편집:** 터미널 내장 편집기인 `nano` 또는 `vim`을 사용하거나, GUI 환경에서 `code [파일명]`을 입력해 Visual Studio Code로 편집합니다.

### 2.4. 파일 권한(Permission)과 `chmod`

- **권한 개념:** 리눅스의 모든 파일은 소유자(User), 그룹(Group), 기타 사용자(Others)별로 **읽기(r, 4), 쓰기(w, 2), 실행(x, 1)** 권한을 가집니다.
- **권한 변경 (`chmod`):** `chmod 755 script.sh` 명령을 통해 소유자에게는 모든 권한(7), 나머지는 읽기/실행 권한(5)만 부여하거나, `chmod +x script.sh`로 실행 권한을 직관적으로 추가할 수 있습니다.

### 2.5. 실행파일, 셸 스크립트와 `source` 명령어

- **실행파일/셸 스크립트:** 실행 권한(`x`)이 부여되어 독자적으로 동작할 수 있는 프로그램 또는 명령어들의 집합 파일(`.sh`)입니다.
- **`source` 명령어:** 스크립트를 독립된 자식 셸이 아닌, **현재 열려있는 터미널(부모 셸) 환경 내에서 직접 실행**하는 명령어입니다. ROS2의 환경 변수 스크립트(`setup.bash`)를 현재 세션에 즉시 적용할 때 필수적으로 사용됩니다.

### 2.6. 사용자 권한: 슈퍼유저와 `sudo`

- **슈퍼유저 (root):** 시스템 전반의 모든 파일을 수정, 삭제, 제어할 수 있는 무제한의 권한을 가진 최고 관리자 계정입니다.
- **`sudo` (Superuser Do):** 일반 사용자가 일시적으로 슈퍼유저의 권한을 대행하여 보안상 제한된 시스템 명령을 실행할 수 있게 해주는 접두 명령어입니다.

### 2.7. 우분투 패키지 관리 (`apt`)

우분투는 `apt`(Advanced Package Tool)를 통해 패키지의 의존성을 자동으로 해결하고 소프트웨어를 관리합니다.

- `sudo apt update`: 원격 저장소의 패키지 목록 동기화
- `sudo apt install [패키지명]`: 특정 소프트웨어 설치

### 2.8. 홈 디렉토리 (Home Directory)

- 각 사용자 계정에게 온전한 소유권이 할당된 개인 작업 공간으로, 기호 **`~`** (물결표)로 표시됩니다.
- 슈퍼유저의 권한(`sudo`) 없이도 자유롭게 파일을 읽고 쓰고 지울 수 있는 안전한 영역입니다.

## 3. 프로그램 실행 절차 및 결과 검증

### 3.1. 사전 준비 (디렉토리 생성)

작성할 프로그램이 루트(`/`) 하위의 `/test` 디렉토리에 접근해야 하므로, 관리자 권한을 사용하여 디렉토리를 선행 생성합니다.

```bash
$ sudo mkdir /test
[sudo] password for user:
```
