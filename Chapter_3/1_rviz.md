# [과제 3-1] RViz2 및 TF(Transform Frames) 조사 보고서

## 1. 실습 환경 구성 및 로봇 모델 확인 (RViz2)

### 1.1. 패키지 설치 및 실행

- **패키지 설치:** `sudo apt install ros-humble-urdf-tutorial` 명령을 통해 실습용 패키지를 설치 완료함.
- **실행 명령어:**
  `ros2 launch urdf_tutorial display.launch.py model:=/opt/ros/humble/share/urdf_tutorial/urdf/08-macroed.urdf.xacro`
- 위 명령어를 통해 시뮬레이터인 RViz2와 로봇의 관절 상태를 제어할 수 있는 `Joint State Publisher` 팝업 UI가 정상적으로 실행됨을 확인.

### 1.2. 샘플 로봇 모델 구조 파악

- RViz2의 `Displays > RobotModels > Links` 항목의 체크박스를 활성화/비활성화하며 로봇의 계층적 부품 구조를 확인.
- 샘플 로봇은 `base_link`를 중심으로 다리, 바퀴, 머리 등의 링크(Link)들이 조인트(Joint)를 통해 서로 연결된 형태임.
- `Joint State Publisher`의 슬라이더 바를 조작하여 각 조인트의 상태 값을 변경하면, RViz2 화면 상에서 로봇의 부품들이 실시간으로 회전하거나 이동하는 것을 관찰할 수 있음.

## 2. TF (Transform Frames) 주요 개념 및 필요성

### 2.1. TF (Transform Frames) 란?

ROS2 시뮬레이션 및 실제 로봇 환경에서 **로봇의 각 링크(부품) 별 좌표계 간의 위치 및 회전 관계를 표현하고 시간에 따라 관리하는 시스템**임.

### 2.2. TF를 구성하는 주요 3가지 개념

1. **좌표계 (Coordinate Frame):** 로봇의 각 링크(부품) 또는 특정 환경 기준점에 부착되어 정의된 고유한 3차원 축. (단순한 위치 값이 아닌 기준계를 의미함)
2. **변환 (Transform):** 한 좌표계에서 다른 부모/자식 좌표계로 넘어가기 위한 수학적 관계식 및 수치 오프셋.
3. **트리 (Tree):** 로봇 내에 존재하는 수많은 좌표계들과 그들 간의 변환 관계를 부모-자식 형태의 계층(네트워크) 구조로 연결하여 추적 가능하게 만든 형태.

### 2.3. TF의 필요성

로봇의 각 부품은 고유한 좌표계를 가지고 있으며, 모든 부품은 관절을 통해 물리적으로 연결되어 움직임. 따라서 센서(예: 카메라, 라이다)가 측정한 데이터를 로봇의 몸체나 주행 공간 기준으로 재해석하거나, 로봇의 전체 동작을 유기적으로 제어(동기화)하기 위해서는 **부품 간의 위치 및 동작을 변환해 주는 좌표계, 변환, 트리 구조가 필수적**임.

### 2.4. 좌표계 변환 시 계산되는 물리량

한 좌표계에서 다른 좌표계로 변환할 때는, 3차원 공간상의 거리를 의미하는 **위치(Position / Translation)** 물리량뿐만 아니라, 부품이 어느 방향을 바라보며 틀어져 있는지를 나타내는 **회전 방향(Rotation / Orientation)** 물리량에 대한 계산이 반드시 포함됨.

## 3. RViz2에서의 TF 시각화 및 좌표축 확인

### 3.1. TF 프레임 시각화

- `Displays > RobotModels`의 선택을 취소하고 `Display > TF`를 선택하면 로봇 외형 대신 좌표계 축들이 화면에 시각화됨.
- `TF > Frames` 항목에서 모든 선택을 취소하고 `base_link`만 체크하면, 베이스 링크 기준의 단일 좌표축 정보만 깔끔하게 확인할 수 있음.

### 3.2. RViz2 시뮬레이터 상의 축 표현 방식 (RGB 색상)

화면에 표시되는 좌표계 축의 붉은색, 파란색, 녹색 선은 3차원 공간의 3축을 직관적으로 나타냄.

- **붉은색 선 (Red):** $X$축 (일반적으로 로봇의 정면, 전진 방향)
- **녹색 선 (Green):** $Y$축 (일반적으로 로봇의 좌측 방향)
- **파란색 선 (Blue):** $Z$축 (일반적으로 로봇의 상단, 하늘 방향)

### 3.3. RViz2에서 TF Tree 확인 방법

- **GUI 활용:** RViz2 좌측 `Displays` 패널 내 `TF` 항목의 `Frames` 목록을 펼치면, 들여쓰기 된 구조를 통해 어떤 프레임이 부모이고 자식인지 트리(Tree) 계층을 시각적으로 파악할 수 있음.
- **CLI 활용 (추가 사항):** 터미널에서 `ros2 run tf2_tools view_frames` 명령어를 실행하면 시스템 내부적으로 동작 중인 전체 TF 트리의 연결 상태를 PDF 파일 형태로 추출하여 확인할 수 있음.
